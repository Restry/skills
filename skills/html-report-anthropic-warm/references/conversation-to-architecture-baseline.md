# 从多轮架构对话生成“已确认基线”HTML

适用于：用户经过多轮纠偏、官方文档核验和方案收敛后，要求把共识整理成可评审、可发布的 HTML。

## 核心原则

HTML 不能只是聊天摘要。它必须把对话压缩成一份**决策基线**，清楚区分：

1. **用户已确认的产品决策**：入口、角色、目标、交付标准。
2. **官方文档已证明的框架能力**：附权威链接，不把推断写成事实。
3. **架构建议**：说明为什么这样组合。
4. **仍待运行验证的假设**：明确进入 Phase 0 spike，不用文档推断替代执行证据。
5. **被否定的旧理解**：保留一张显式对比表，防止旧 mental model 回流。

最新一次用户纠偏优先级最高；旧方案与最新结论冲突时，不做折中拼接。

## 写前决策账本

先在草稿中整理五栏，确认无冲突后再写 HTML：

| 栏 | 要记录什么 |
|---|---|
| Confirmed | 用户明确拍板的内容 |
| Official facts | 官方文档或当前实现直接证明的事实 |
| Architecture | 基于事实做出的设计选择 |
| Unverified | 需要 spike / E2E 才能确认的组合能力 |
| Rejected | 用户已否定的入口、流程、抽象或表达方式 |

禁止把 `Rejected` 内容留在 Hero、流程图、KPI、里程碑或界面示意里。

## 推荐的五章结构

### 01 · 最终架构结论

- Hero 一句话写清核心链路。
- 画“输入 → 规划 → 校验/执行 → 交付”关系图。
- 放一张“被否定的思路 vs 已确认方案”对比表。
- 先讲业务含义，再出现框架术语。

### 02 · 用户与管理员视角

- 用户视角：从进入页面到拿到结果，按时间顺序描述。
- 管理员视角：模型、版本、运行记录、调试工具分别维护什么。
- 不从类名、包名、Executor、数据库表开始。

### 03 · 运行机制

- 明确哪个组件理解需求、哪个组件生成计划、哪个组件执行。
- 若采用动态计划，给一个精简但机器可执行的 PlanSpec 示例。
- 讲清计划验证、依赖、并行、Step、checkpoint 和修复路径。
- 明确“动态 Plan”不等于“LLM 动态生成并执行源代码”。

### 04 · 交付契约

- 列出固定输出结构。
- 列出确定性通过条件。
- 单独列“不能作为完成依据”的信号，例如 Todo 完成、模型口头声明、占位资产。

### 05 · 落地与风险

- 前端、后端、存储、调试工具各一张卡。
- 按 Phase 0 技术探针 → 领域契约 → 运行闭环 → 联调 → 真实验收排列。
- Experimental / Preview API 必须写版本锁定、适配层和升级回归。

## 动态计划类架构的表达模板

当方案核心为“需求 → Planner → 动态计划 → Workflow 执行”时，必须把质量责任拆开：

- Planner 决定**做对哪些事**。
- Workflow 保证**按合法计划执行与恢复**。
- Skills 提供**领域方法**。
- Tools 执行**真实动作**。
- Gate 证明**产物合格**。

PlanSpec 至少展示：`id`、`objective`、`dependsOn`、`parallelGroup`、`skillHints`、`allowedTools`、`deliverableType`、`acceptanceCriteria`。

Plan Validator 至少检查：

- ID 唯一、依赖存在、无环。
- 输入输出类型可衔接。
- 并行组安全。
- Skill / Tool 来自允许列表。
- 强制交付任务可达。
- 计划通过后冻结版本；恢复时不得重新生成不同计划。

## 内容验证清单

- [ ] Hero 能在一屏内读懂，不拆中文词组。
- [ ] 章节标题中文优先，英文只作术语或副标。
- [ ] 有用户和管理员两种视角。
- [ ] 有旧理解 vs 新方案的显式对比。
- [ ] 官方事实带权威链接。
- [ ] Experimental / 未证实组合没有被写成“已支持”。
- [ ] 固定交付标准与动态执行路径分开描述。
- [ ] 终态由确定性 Gate 定义，不信任模型口头完成。
- [ ] 发布前跑 emoji 扫描、禁用称呼词扫描、浏览器视觉检查。
- [ ] 公网 HTTP 200，下载文件与本地 hash 一致。

## 反模式

- 框架先行：先讲 Agent / Workflow 定义，再让读者猜产品怎么用。
- 把用户具体需求写死成场景入口或固定节点。
- 用长 Todo 代替可保存、可校验的执行计划。
- 把“原生 Python 控制流”误写成“框架自动生成动态代码”。
- 把官方支持的单项能力直接推断成组合能力已跑通。
- 在同一份 HTML 中混留多个互相冲突的架构版本。
